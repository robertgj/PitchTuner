Detailed Description

template<typename StoredType, typename InputType = StoredType, typename OutputType = StoredType, typename IC = saCircBuffCopyByteWise<StoredType>, typename OC = IC, bool DoOverRunException = true, bool DoUnderRunException = true>
class SimpleAudio::saCircBuff< StoredType, InputType, OutputType, IC, OC, DoOverRunException, DoUnderRunException >

SimpleAudio circular buffer class.

Implementation of a lock-free, thread-safe circular buffer.

  • m_back is an atomic variable that indicates the next availabe free sample storage location in the circular buffer and is only modified by Insert() after the copy is complete. The internal size of the buffer is set to one greater than that requested so that the storage at m_back is always empty.
  • m_front is an atomic variable that indicates the start of sample storage in the circular buffer and is only modified by Extract() after the copy is complete.
  • Insert() reads m_front once before calculating the space available for writing.
  • Extract() reads m_back once before calculating the number of samples available for reading.
  • the samples available for reading from the circular buffer lie between m_front and m_back. If m_back < m_front then the sample storage wraps around the end of the buffer memory back to the start.
  • if m_front == m_back then the buffer is empty.
  • Clear() sets both m_front and m_back to 0. You should not call Clear() from a producer or consumer thread!

See: "Writing Lock-Free Code: A Corrected Queue", H. Sutter, Dr Dobb's Journal, September 29, 2008.
